FLUJO DE NAVEGACIÓN:
  showSelect (roster) → PatientSelectScreen
  !welcomed (primer run) → WelcomeScreen
  screen === 'home' → HomeScreen
  screen === <activityId> → <ActivityScreen>

ACTIVIDADES REGISTRADAS (ACTIVITY_LABELS + ACTIVITY_SCREENS):
  minimal-pairs       → 'Palabras Similares'
  build-word          → 'Armar Palabras'
  listen              → 'Escucha Atento'
  syntax              → 'Completar Frases'
  semantic            → 'Semántica'
  narrative           → 'Ordenar Historia'
  pragmatic           → 'Inferencias'
  rhyme               → 'Rimas'
  point-image         → 'Señala la Imagen'
  category            → '¿Cuál no pertenece?'
  follow-instruction  → 'Sigue la Instrucción'
  communicative-intent→ '¿Para qué sirve?'
  (total: 12 actividades)

PANTALLAS NO-ACTIVIDAD (no en ACTIVITY_SCREENS, no miden tiempo):
  progress, session-history, results

SISTEMA DE PUNTUACIÓN (finishActivity):
  score == total        → 3 estrellas
  score >= total * 0.6  → 2 estrellas
  score < total * 0.6   → 1 estrella
  Guarda entry en sessionHistory (contexto + roster localStorage)
  Mide duration: Date.now() - activityStartRef.current (en segundos)

PROPS DE PANTALLAS DE ACTIVIDAD:
  - Con onBack:  MinimalPairsScreen, BuildWordScreen, SemanticScreen, ListenScreen,
                 SyntaxScreen, PragmaticScreen, NarrativeScreen
  - Sin onBack:  RhymeScreen, PointImageScreen, CategoryScreen, FollowInstructionScreen,
                 CommunicativeIntentScreen
  NOTA: Las pantallas sin onBack usan onFinish directamente para volver
        (llamando onFinish(score, items.length) desde el botón ← del header).
        Son pantallas más nuevas con diseño propio (inline styles, no className).

PATRÓN CONSISTENTE en todas las actividades:
  finishActivity(s, t, activityId) donde s=score, t=total

BUG POTENCIAL — WelcomeScreen:
  setLevelByAge(age) pero la función en PatientContext puede ser setLevel o setLevelByAge.
  Requiere verificar que PatientContext expone exactamente 'setLevelByAge'.

SECCIONES (LANGUAGE_LEVELS):
  fonetico       (#4aab8a): minimal-pairs, build-word, rhyme
  lexico         (#7c6bb0): listen, semantic, point-image, category
  morfosintactico(#e07a5f): syntax, narrative, follow-instruction
  pragmatico     (#e8a020): pragmatic, communicative-intent

LÓGICA DE DISPONIBILIDAD (isActivityAvailable):
  Lee desde nivel (object de levels.js), NO desde JSON directo.
  Rutas de datos:
    minimal-pairs      → level.fonologia.minimalPairs
    build-word         → level.fonologia.buildWords
    listen             → level.semantica.listen
    semantic           → level.semantica.opposites || level.semantica.definitions
    syntax             → level.morfosintaxis.connectors
    narrative          → level.morfosintaxis.narrativeSequence
    pragmatic          → level.pragmatica.inferences
    rhyme              → level.rhymes          (raíz del nivel)
    point-image        → level.pointImages     (raíz del nivel)
    category           → level.categories      (raíz del nivel)
    follow-instruction → level.instructions    (raíz del nivel)
    communicative-intent→ level.communicativeIntents (raíz del nivel)

  hasContent(obj):
    null/undefined → false
    Array → obj.length > 0
    Object → algún valor es array con length > 0
    (Soporta tanto arrays planos como {inicial,intermedio,avanzado})

PROGRESO DE LA SEMANA:
  Los porcentajes (72%, 60%, 40%, 55%) son VALORES HARDCODED en LANGUAGE_LEVELS,
  no calculados desde sessionHistory.
  BUG / DEUDA: No reflejan progreso real del paciente.
  Sugerencia futura: calcular desde patient.sessionHistory por componente.

SUGERENCIAS POR DIAGNÓSTICO:
  tel         → pares mínimos + conectores
  tl_tea      → inferencias + contexto social
  tl_tdah     → discriminación auditiva + semántica + escucha atenta
  tl_tea_tdah → inferencias simples + escucha atenta

CONTEXTO CONSUMIDO: patient, level, stimulusConfig (NO estimulusSettings)
  stimulusConfig → configuración de diagnóstico (color, label), no ajustes de estímulo.

3. PragmaticScreen.jsx — INFERENCIAS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

DATOS:
  Fuente: getContent(patient.levelId).inferences[difficulty]
  Fallback: .inferences.inicial si el nivel de dificultad no existe
  Default exerciseCount: 12

FORMATO DE ÍTEM:
  { situation: string, emoji?: string, question: string, correct: string, options: string[] }
  Options son strings planos (igual que communicativeIntentScreen).

MECÁNICA:
  1. Shuffle de opciones en useEffect([idx]) con Math.random()
  2. handleAnswer → playFeedback → setTimeout(1000) → muestra feedback
  3. setTimeout(800) adicional → muestra botón "Siguiente →"
  4. Botón "Siguiente" llama nextAction.current()
  Patrón: requiere acción manual del usuario para avanzar (NO auto-advance).
  Contrasta con CommunicativeIntentScreen que auto-avanza a los 1400ms.

reducedOptions: Usa una IIFE para filtrar máximo 2 opciones (correcto + 1 incorrecto).
  Lógica: shuffledOptions.find(o => o !== current?.correct) — correcto.

PROPS: onFinish(score, total), onBack
CONTEXTO: patient, level, estimulusSettings

ESTIMULUESSETTINGS USADOS:
  exerciseCount['pragmatic']   → n° ejercicios (default 12)
  animationsEnabled            → playFeedback + clase no-anim
  backgroundElements           → whiteBg (background: white)
  largerText                   → instrSize (18px vs 15px)
  reducedOptions               → max 2 opciones
  simplifiedInstructions       → texto de instrucción

NO USADOS (a diferencia de otras pantallas):
  slideTransitionDelay  → ya removido (bug fix previo)
  simultaneousAudioVisual → no tiene audio, no aplica

DIFERENCIA VISUAL con pantallas nuevas:
  Usa clases CSS (className="screen", "activity-header", "game-area", "check-btn")
  en lugar de inline styles. Pantallas nuevas usan 100% inline styles.

4. MinimalPairsScreen.jsx — PARES MÍNIMOS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

DATOS:
  Fuente: getContent(patient.levelId).minimalPairs[difficulty]
  Fallback: .minimalPairs.inicial
  Default exerciseCount: 12

FORMATO DE ÍTEM:
  { word: string, emoji: string, distractor: { word: string, emoji: string } }
  SIEMPRE 2 opciones: target + distractor. reducedOptions no tiene efecto aquí.
  (Comentario en código lo confirma: "MinimalPairs siempre son 2 opciones")

AUDIO: speak() definida a nivel módulo (fuera del componente).
  speak(text, rate=0.82, pitch=1.05) — usa window.speechSynthesis
  Usa rate=0.7 cuando simultaneousAudioVisual es FALSE (más despacio para
  la condición "primero mira, luego escucha").

MECÁNICA:
  1. handlePlay() → speak(current.word) — botón ▶ visible
  2. handleAnswer(word) → speak(word) + playFeedback
  3. Mismo patrón de setTimeout que PragmaticScreen (1000ms + 800ms)
  4. Botón "Siguiente →" manual

FORMATO DE ÍTEM (historia):
  { title: string, frames: [{ emoji, text }], correctOrder: number[] }
  frames[i].originalIndex se asigna dinámicamente al barajar.
  correctOrder es array de índices del orden correcto.

MECÁNICA ÚNICA — Intercambio por toque:
  El usuario toca 2 tarjetas para intercambiarlas (no drag-and-drop).
  selected: índice de la tarjeta actualmente seleccionada.
  handleCardClick(position):
    si no hay seleccionada → seleccionar esta
    si es la misma → deseleccionar
    si es diferente → swap en array cards

  handleCheck(): verifica que cards[i].originalIndex === story.correctOrder[i]
  handleReset(): re-mezcla sin cambiar idx (reactiva baraja)
