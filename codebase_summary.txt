RESUMEN DE ARCHIVOS — AuraPlay / NexiaPlay
==========================================
Fecha: 2026-04-21

────────────────────────────────────────
1. src/App.jsx
────────────────────────────────────────
Punto de entrada principal de la app. Gestiona la navegación entre pantallas con
estado local `screen` (string). Contiene:

- ACTIVITY_LABELS: mapa de IDs de actividad a etiquetas legibles.
- ACTIVITY_SCREENS: Set de pantallas que cuentan como actividades (inician timer).
- WelcomeScreen: formulario de alta de primer paciente (nombre, edad en meses,
  diagnóstico). Llama a savePatient() y loadPatient() del contexto.
- App (componente raíz):
  - Lógica de inicio: si hay pacientes en roster → muestra PatientSelectScreen;
    si no hay ninguno → muestra WelcomeScreen.
  - goTo(screenName): cambia pantalla y registra timestamp de inicio.
  - finishActivity(score, total, activityId): calcula estrellas (1/2/3),
    construye entry de sesión, actualiza contexto Y persiste en roster de pacientes.
  - Renderiza condicionalmente cada pantalla según `screen`.

Persistencia dual: localStorage['auraplay_patient'] (contexto activo) +
localStorage['auraplay_patients'] (roster general).

────────────────────────────────────────
2. src/screens/TherapistPanel.jsx
────────────────────────────────────────
Panel modal del terapeuta. Requiere PIN para acceder (default: '1234').
Estructura principal:

- PinScreen: teclado numérico 4 dígitos. Error visual si PIN incorrecto.
- TherapistPanel (exportado): modal bottom-sheet tras autenticación.
  Vistas internas: 'menu' | 'new' | 'search' | 'config'
  - menu: muestra paciente activo, acciones (nuevo / buscar / configurar / guardar).
  - new → NewPatientForm: campos RUT (con validación dígito verificador),
    nombre, teléfono, tutor, fecha nacimiento, diagnóstico, nivel inicial.
  - search → SearchPatient: busca en roster, muestra lista; al seleccionar
    abre PatientSummary con resumen clínico completo.
  - config → ConfigPanel (4 tabs):
    - Paciente: nombre, diagnóstico, foto, perfil sensorial, hitos, cambiar PIN,
      reset paciente.
    - Nivel: selector de nivel cronológico + niveles por componente lingüístico
      (fonológico / léxico / morfosintáctico / pragmático: inicial/intermedio/avanzado),
      + preview de ejercicios disponibles por actividad con input de cantidad.
    - Estímulos: toggles visual/auditivo + sliders de velocidad (wordSpeakDelay,
      slideTransitionDelay).
    - Registro (SessionTab): formulario de nota clínica (fecha, duración, tests,
      texto 300 palabras), análisis lingüístico con debounce 800ms via analyzeText(),
      historial últimas 3 sesiones.

Sub-componentes:
- MilestonesSection: hitos del desarrollo por nivel (N1–N7), estado cíclico
  (no_iniciado → en_proceso → logrado).
- SensoryProfile: badges read-only del perfil sensorial actual.
- StimulusToggle: toggle switch reutilizable.
- ChangePinSection: permite cambiar el PIN de 4 dígitos.

Constantes de estilo compartidas al final: labelStyle, inputStyle,
primaryBtnStyle, backBtnStyle, errorStyle.

────────────────────────────────────────
3. src/screens/PatientSelectScreen.jsx
────────────────────────────────────────
Pantalla de selección de paciente al inicio de la app (cuando ya hay roster).

- Lista "RECIENTES": top 5 pacientes ordenados por updatedAt.
- Buscador: filtra por nombre o RUT via searchPatients().
- Al seleccionar → ConfirmCard (bottom-sheet): muestra resumen del paciente
  (nivel, sesiones, estrellas, última actividad, niveles por componente).
- Botón "Nueva sesión" → NewPatientForm interna (misma lógica que TherapistPanel
  pero con UI propia de esta pantalla).

Funciones helper locales (duplicadas de TherapistPanel):
  formatRut, validateRut, calcAgeMonths, fmtDate.

Nota: hay duplicación de helpers y NewPatientForm entre PatientSelectScreen.jsx
y TherapistPanel.jsx (potencial refactor futuro).

────────────────────────────────────────
4. src/utils/audioFeedback.js
────────────────────────────────────────
Utilidad de feedback auditivo usando Web Audio API.

Exporta: playFeedback(type, enabled)
  - enabled=false → no hace nada (silencio).
  - type='correct': secuencia ascendente 523Hz → 659Hz → 784Hz (3 notas sine,
    0.13s de separación, ganancia 0.28 con fade exponencial).
  - type=cualquier otra cosa: tono descendente 400Hz → 300Hz en 0.3s,
    ganancia 0.18 con fade.
  - Try/catch silencioso si AudioContext no está disponible.

────────────────────────────────────────
ARQUITECTURA GENERAL
────────────────────────────────────────
- Framework: React + Vite (puerto 5173).
- Navegación: state machine manual en App.jsx (sin router).
- Estado global: PatientContext (usePatient hook) — paciente activo, nivel,
  configuración de estímulos, historial de sesión.
- Persistencia: localStorage.
  - 'auraplay_patient': paciente activo en sesión.
  - 'auraplay_patients': roster completo de pacientes.
  - 'auraplay_pin': PIN del terapeuta.
- Diagnósticos: configurados en STIMULUS_CONFIG (data/levels.js).
  Tipos vistos: 'tel' (TEL), y al menos uno más.
- Niveles: N1–N7 (data/levels.js), con rango de edad por nivel.
- Actividades disponibles (12): minimal-pairs, build-word, listen, syntax,
  semantic, narrative, pragmatic, rhyme, point-image, category,
  follow-instruction, communicative-intent.
